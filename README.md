# College Schedule App 📚

Мобильное приложение для просмотра расписания пар в колледже. Бэкенд разбирает Excel-таблицу с расписанием и отдаёт данные через REST API. Мобильное приложение на React Native отображает расписание на сегодня, на неделю и расписание звонков.

## Функциональность

| Экран | Описание |
|---|---|
| 📅 Расписание на сегодня | Автоматически определяет день недели и показывает пары |
| 📖 Расписание на неделю | Полное расписание по всем дням сразу |
| 🔔 Расписание звонков | Фиксированное время начала и конца каждой из 8 пар |
| ⚙️ Настройки | Загрузка новой Excel-таблицы расписания с телефона |

Дополнительно на экранах «Сегодня» и «Неделя» отображаются:
- 🍽️ Время обеда (11:25–12:05, если есть в таблице)
- 🧹 Уборка после занятий

## Установка и запуск

### Бэкенд

**Требования:** Python 3.13+

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Бэкенд запустится на `http://0.0.0.0:8000`. При первом запуске папка `uploads/` создаётся автоматически. Для работы нужно загрузить таблицу расписания через эндпоинт `/upload` или положить файл вручную как `backend/uploads/active.xlsx`.

---

### Мобильное приложение

**Требования:** Node.js, установленное приложение [Expo Go](https://expo.dev/go) на телефоне.

**1. Перейдите в папку и установите зависимости:**

```bash
cd mobile
npm install
```

**2. Укажите адрес бэкенда в `src/config/api.ts`:**

```ts
// Для локальной разработки — IP вашего компьютера:
export const API_URL = "http://192.168.X.X:8000";

// Для продакшена (Railway):
export const API_URL = "https://college-schedule-app-production.up.railway.app";
```

> Телефон и компьютер должны быть в одной Wi-Fi сети при локальной разработке.

**3. Запустите приложение:**

```bash
npx expo start
```

Отсканируйте QR-код в Expo Go или нажмите `a` для запуска на Android-эмуляторе.

## API

Базовый URL: `https://college-schedule-app-production.up.railway.app`

### `GET /today` — расписание на сегодня

Автоматически определяет текущий день недели и возвращает пары.

**Ответ:**
```json
{
  "day": "ПОНЕДЕЛЬНИК",
  "lessons": [
    {
      "number": 1,
      "time": "8:30-9:10",
      "subject": "Математика",
      "room": "205"
    }
  ],
  "info": [
    { "type": "lunch", "text": "Столовая", "time": "11:25-12:05" },
    { "type": "cleaning", "text": "Уборка кабинета", "time": null }
  ]
}
```

    { "number": 2, "time": "9:15-9:55" },
    { "number": 3, "time": "10:00-10:40" },
    { "number": 4, "time": "10:45-11:25" },
    { "number": 5, "time": "12:05-12:45" },
    { "number": 6, "time": "12:50-13:30" },
    { "number": 7, "time": "13:35-14:15" },
    { "number": 8, "time": "14:20-15:00" }

---

### `POST /upload` — загрузка новой таблицы

Принимает `multipart/form-data` с полем `file`. Сохраняет файл как `uploads/active.xlsx` — он сразу становится активным для всех последующих запросов.

```bash
curl -X POST https://.../upload -F "file=@schedule.xlsx"
```

**Ответ:**
```json
{ "status": "ok", "file": "schedule.xlsx" }
```

## Формат Excel-таблицы

Парсер читает файл без заголовков (`header=None`) и обращается к колонкам по индексу:

| Колонка (0-based) | Содержимое |
|---|---|
| 1 | День недели (`ПОНЕДЕЛЬНИК`, `ВТОРНИК`, ...) |
| 2 | Номер пары (1–8) |
| 3 | Время пары (например, `8:30-9:10`) |
| 30 | Название предмета / группы |
| 31 | Номер кабинета |

**Важные детали:**
- Строка с названием дня определяет начало блока этого дня. Блок заканчивается при появлении следующего непустого значения в колонке 1.
- Строки, где предмет содержит `уб` — интерпретируются как уборка.
- Строки, где предмет содержит `столовая` или `соловая` — интерпретируются как обед.
- Пары с номером вне диапазона 1–8 или с пустыми полями игнорируются.

## Сборка APK (Android)

### 1. Регистрация и вход

```bash
npx expo login
```

### 2. Установите EAS CLI

```bash
npm install -g eas-cli
eas --version
```

### 3. Инициализация (один раз на проект)

```bash
cd mobile
eas init
```

### 4. Сборка APK

```bash
eas build -p android --profile preview
```

Профиль `preview` настроен в `eas.json` — собирает `.apk` для внутреннего распространения.

### 5. Установка

- Сборка занимает **5–20 минут** на серверах Expo
- После завершения появится ссылка для скачивания APK
- Перед установкой разрешите установку из неизвестных источников в настройках Android

> Для публикации в Google Play используйте профиль `production`:
> ```bash
> eas build -p android --profile production
> ```

## Деплой бэкенда

Бэкенд задеплоен на [Railway](https://railway.app). Для деплоя своей версии:

1. Создайте аккаунт на railway.app
2. Создайте новый проект из GitHub репозитория
3. В настройках укажите `Start Command`:
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```
4. Railway автоматически установит зависимости из `requirements.txt`

После деплоя обновите `API_URL` в `mobile/src/config/api.ts`.
